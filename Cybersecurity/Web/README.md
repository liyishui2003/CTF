## 进阶路线

```
CTF-01                     CTF-02                      CTF-03                      CTF-04                      CTF-05                      CTF-06                      CTF-07
浏览器层面                  HTTP 协议层面               代码执行层面                内存层面                   编码层面                   页面结构层面                请求篡改层面
┌─────────────┐           ┌─────────────┐             ┌─────────────┐            ┌─────────────┐            ┌─────────────┐            ┌─────────────┐            ┌─────────────┐
│ 改 Cookie    │   ──→    │ 加 Header   │    ──→      │ 插 Payload  │    ──→     │ 下文件搜    │    ──→    │ 看 cookie   │   ──→     │ 翻页面    │    ──→   │ 删参数  │
│ 替换 session │           │ X-Dev-Access│             │ open('flag')│            │ heapdump   │            │ 解 Base64  │            │ 扒属性    │           │ 绕过 OTP │
│ 无需理解原理  │           │ 需要理解请求 │             │ 需要理解     │            │ 需要知道    │            │ 需要知道    │            │ 需要理解   │           │ 需要理解  │
│              │           │ 与编码      │             │ Python 对象  │            │ flag 格式   │            │ 编码 ≠ 加密 │            │ HTML 结构  │           │ 服务端校验 │
└─────────────┘           └─────────────┘             └─────────────┘            └─────────────┘            └─────────────┘            └─────────────┘            └─────────────┘
```

**共同本质**：服务端对"信任边界"的假设出了问题。

| 题目 | 服务端信任了什么 | 实际被利用了什么 |
|------|-----------------|-----------------|
| CTF-01 | 信任 session 的有效期 | session 永不过期 → 直接劫持管理员 |
| CTF-02 | 信任 `X-Dev-Access` header | 调试后门暴露 → 任意用户伪装管理员 |
| CTF-03 | 信任用户输入不包含模板语法 | Jinja2 渲染未转义 → 任意文件读取 |
| CTF-04 | 信任诊断接口不会被发现 | 堆快照暴露 → flag 在内存里明文躺着 |
| CTF-05 | 信任 Base64 编码 = 安全 | Cookie 里 Base64 编码 → 任何人都能解 |
| CTF-06 | 信任自定义属性不被发现 | `notify_true` 属性 → 查看源代码就能看到 |
| CTF-07 | 信任 OTP 参数存在且有效 | 删除 `otp` 参数 → 空参数绕过校验 |

## 怎么用这些 writeup

1. **按顺序看** — 难度递增，概念逐步加深
2. **看懂概念链** — CTF-03 的 `.__class__.__mro__.__subclasses__()` 链是 Python 对象模型的核心，值得反复理解
3. **动手复现** — 每题都附带完整的复现步骤
4. **关联到工程** — SSTI 防御就是中间件/方法论里的内容，不是孤立的知识
